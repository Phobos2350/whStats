Recent Updates

2016-06-20 - v1.8 - Entity Stats

First version of Entity Stats:
- View raw stats per entity (corp/alliance) per Timezone period (GMT)
- Click on each entity to reveal graph-summaries of ship doctrine stats

2016-06-16 - v1.7 - Slim Framework

Re-wrote the entire application to use Slim Framework 3, this brings:
- Pretty URLs
- Better handling of GET/POST in relation to passing data around the site
- An API interface for getting raw data from the site, to be described soon!

2016-06-14 - v1.6 - Monthly Stats

Added Monthly stats, resets every month.
Made all pie/doughnuts smaller

2016-06-14 - v1.5 - Cap/Logi Breakdowns

Added breakdowns of Cap and Logistic Cruiser usage over each WH class.

2016-06-13 - v1.4 - Back-End Improvements

Improved the accuracy of data pulls.

2016-06-09 - v1.3 - Daily and Weekly Stats

Added two whole new pages for daily stats, reset every night at midnight GMT, and weekly, reset at midnight GMT every Sunday morning!

2016-06-09 - v1.2 - Hour Kill Hotfix

Wasn't tracking new hour kills we've already seen as they came in, resulting in multiple counts in some cases

2016-06-08 - v1.1 - Kill Logic Review

Kills are now pulled in a more consistent manner in the hope that kills added 'late' are picked up and recorded (e.g. when an hour has been pulled but new kills are added from a slow API/CREST update)
In addition the graphs should update to reflect these changes when new kills are detected in the last hour 'block'.
Added in some missing structure IDs.

2016-06-08 - v1.0 - Tweaks/Bugfixes

Tweaked 'Fleet Fight' classification to at least 5 per side to better count 'fights' on most scales
Data is now current from 206-06-03 @ 10:00 onwards (older data is hard to reach, we use zkills w-space/page/100 as a hard limit for populating old data)
However all data from this point onwards (8th June 2016) will be tracked and updated hourly
A few bugs still exist and the auto hourly-refresh can just stop working on some browsers or if a particular kill scrape takes longer than usual
General Fixes: - Fighter kills are now detected and discarded, we are only interested in ship, structure and citadel kills
- Added tracking for special edition and rare faction ships, Utu, Etana etc. Classed as generic 'Faction'
- Fixed infinite loop on a case where the first kill processed was of an unknown type (what???)
- Added error handling and catch blocks for occasions where things go wrong
- Displays a 'No Data' prompt on graphs that have nothing to display (looking at you C6s...)
- Sat through 63 thousand lines of returned data and almost choked on a triple chocolate muffin while doing so

2016-06-07 - v0.9 - CSS/Layout

Site styling updated, graphs made less humungous on large screens (finally). Feedback appreciated

2016-06-07 - v0.8 - Fleet Fights v2

Improved the fleet fight calculation. Now all kills are cross-checked with a battlereport through some hacky HTML scraping
All kills involving 8 or more people per side show up as a full 'Fleet Fight' whilst those with 7 or less are classed as a 'Small Fight'
While again not 100% true 100% of the time, this should suffice for detecting most legit 'Fights'!

2016-06-07 - v0.7 - Fleet fights

Added a (rough) estimate to number of 'big' and 'small' fights in each class
This is calculated based on a simple count of number of involved people on a kill.
If there are more than 10 entities, then class it as a fleet fight!
This feature will be improved in the future to take more factors into account as this approach is realtively naive

2016-06-06 - v0.6 - ISK Averages

Added ISK total and average displays per class

2016-06-06 - v0.5 - Citadels

Added Citadel Kill Hourly and Total tracking

2016-06-06 - v0.4 - Doughnuts

Added doughnut charts. Yep. Doughnuts.

2016-06-06 - v0.3 - Chartjs

Switched to Chart.js library, reverted from stacked bars to side-by-side

2016-06-06 - v0.2 - Ship Types

Adding ship types to charts and breakdowns according to generalised classes
